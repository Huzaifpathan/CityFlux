1. Splash Screen

🚀 App Launch Screen
📁 ui/splash/SplashScreen.kt

Purpose

First screen when app opens. Checks authentication status and routes to appropriate dashboard.

Flow

App Open ↓ Logo Animation Display (2-3 seconds) ↓ Firebase Auth Check: Is user logged in? │ ├─ NO → Navigate to LOGIN Screen │ └─ YES → Read user role from Firestore │ ├─ role = "citizen" → CITIZEN DASHBOARD ├─ role = "police" → POLICE DASHBOARD └─ role = "admin" → ADMIN DASHBOARD

Features

  • Animated logo display with fade-in effect
  • Firebase Authentication state check
  • Firestore user document read for role detection
  • Automatic navigation based on user role

2. Login Screen

🔐 User Authentication
📁 ui/login/LoginScreen.kt

Login Methods

📧

Email + Password

Traditional login with email and password using Firebase Auth

📱

Phone OTP

10-digit phone number with +91 auto-prefix and 6-digit OTP verification

Complete Flow

Screen Open ↓ Input Field: Enter Email or Phone ↓ System Detection: Email or Phone? │ ├─ EMAIL (contains @) │ ↓ │ Show Password Field │ ↓ │ Tap "Login" Button │ ↓ │ Firebase.signInWithEmailAndPassword() │ ├─ SUCCESS → Read role → Dashboard │ └─ FAIL → Show error message │ └─ PHONE (10 digits) ↓ Auto-add "+91" prefix ↓ Tap "Send OTP" Button ↓ Firebase PhoneAuthProvider.verifyPhoneNumber() ↓ Show OTP Input Field ↓ User enters 6-digit OTP ↓ Verify → Success → Dashboard OTHER OPTIONS: ├─ "Forgot Password?" → ForgotPasswordScreen └─ "Create Account" → RegisterScreen

3. Register Screen

📝 New User Registration
📁 ui/register/RegisterScreen.kt

Registration Form Fields

Field Validation Required
Full Name Minimum 2 characters ✅ Yes
Email Valid email format (user@domain.com) ✅ Yes
Mobile Number Exactly 10 digits ✅ Yes
Password Minimum 6 characters ✅ Yes
Confirm Password Must match password ✅ Yes
Role Selection Citizen or Traffic Police ✅ Yes

Registration Flow

Fill Form Fields ↓ Real-time Validation on Each Field ├─ Error? → Show red error message below field └─ Valid? → Enable Submit button ↓ Tap "Register" Button ↓ Firebase.createUserWithEmailAndPassword() ↓ SUCCESS → Create Firestore Document: { collection: "users", document: "{uid}", data: { name: "User Name", email: "user@email.com", phone: "9876543210", role: "citizen", createdAt: timestamp, lastLoginAt: timestamp } } ↓ Navigate to CITIZEN DASHBOARD

14. Firebase Data Structure

🔥 Database Architecture

Firestore Collections

├─ users/{uid} │ ├─ name: "Shivam Sharma" │ ├─ email: "user@email.com" │ ├─ phone: "9876543210" │ ├─ role: "citizen" │ ├─ profileImageUrl: "https://..." │ ├─ vehicleNumbers: ["MH-20-AB-1234"] │ └─ notifications/{notificationId} │ ├─ title, message, type, priority │ └─ read, pinned, timestamp │ ├─ bookings/{bookingId} │ ├─ userId, parkingSpotId, vehicleNumber │ ├─ bookingStartTime, bookingEndTime, durationHours │ ├─ status, paymentStatus, totalAmount, isPaid │ ├─ qrCodeData │ └─ rating, review │ ├─ parking/{parkingId} │ ├─ address, location (GeoPoint) │ ├─ totalSlots, availableSlots, isLegal │ ├─ parkingType: "free" | "paid" │ ├─ ratePerHour, minDuration, maxDuration │ └─ vehicleTypes: ["car", "bike", "ev"] │ └─ reports/{reportId} ├─ userId, type, title, description ├─ imageUrls[], latitude, longitude ├─ status, priority, isAnonymous ├─ upvoteCount, upvotedBy[], timestamp └─ chat/{messageId}

Realtime Database Nodes

├─ traffic/{areaId} │ ├─ status: "free" | "moderate" | "heavy" | "severe" │ ├─ congestionLevel: 0-100 │ └─ lastUpdated: timestamp │ ├─ parking_live/{parkingId} │ ├─ availableSlots, totalSlots │ ├─ slotsByType: {bike: {}, car: {}, suv: {}} │ ├─ peakHourMultiplier: 1.5 │ └─ lastUpdated: timestamp │ └─ live_users/{userId} ├─ latitude, longitude ├─ role: "citizen" | "police" └─ timestamp
